# oque é COM1 
COM (Communication Port / Porta de Comunicação) é uma interface que permite a transferência de bit um de cada vez. É usada para comunicação RS232 
no Windows é chamado de COM
no Linux é chamado de /dev/ttyS0 (nativo), /dev/ttyS1 Segunda porta serial

| SO      | Nome Porta | Descrição                      |
| ------- | ---------- | ------------------------------ |
| Windows | COM1       | primeira porta serial          |
| Windows | COM2       | segunda porta serial           |
| Linux   | /dev/ttyS0 | primeira porta serial (nativa) |
| Linux   | /dev/ttyS1 | segunda porta serial           |
# Como é feita a comunicação?
Embora Windows e Linux sejam sistemas operacionais diferentes a troca de bits acontece do mesmo jeito pois a comunicação RS232 é baseada em hardware e padrão elétrico

- Camada Física
	RS232 define: 
	- nível de tensão (+12V = 0 lógico, -12V = 1 lógico)
	- formato dos bits (start bit, 8 data bit, paridade, stop bit)
	- direção dos sinais TX e RX, e cabo GND
- Configuração da porta serial 

# RS232 
comunicação é assíncrona. Não há clock de sincronização
- Clock de Sincronização: responsável por gerar um sinal referência que sincroniza todos os outros sinais do sistema
- Comunicação Assíncrona: é aquela que não acontece em tempo real
	**O emissor envia a mensagem e o receptor pode acessá-la e respondê-la quando desejar. O modelo é mais conveniente para quem recebe, porém, requer objetividade, já que pode não ser possível tirar dúvidas no mesmo momento.**
	
já que não há um relógio de sincronização cada BYTE então tem um BIT INICIAL seguido por um BIT FINAL 

# Emuladores de Terminal Serial
eles permitem que o sistema operacional envie e receba dados pela porta serial, como se fosse um terminal de texto remoto
- eles têm como função:
	- abrir a porta serial
	- configurar parâmetros de comunicação
	- permitir digitação e exibição de dados em tempo real
	- facilitar o envio e recepção de arquivos ou comandos serialmente

# Como Conectar os SOs
Solda
---
1) desencape os fios do cabo de rede
2) solde os pinos corretamente no conector DB9
	| Pino | Nome | Função                   |
	| ---- | ---- | ------------------------ |
	| 1    | DCD  | Data Carrier Detect      |
	| 2    | RXD  | Receber dados (entrada)  |
	| 3    | TXD  | Transmitir dados (saída) |
	| 4    | DTR  | Data Terminal Ready      |
	| 5    | GND  | Terra                    |
	| 6    | DSR  | Data Set Ready           |
	| 7    | RTS  | Request To Send          |
	| 8    | CTS  | Clear To Send            |
	// o 9 é dispensável

Comunicação
---
1) conectar o DB9 no Linux e Windows
2) escolher a porta correta:
	1) Linux: /dev/ttyS0...
	2) Windows: COM1, COM3...
3) Abrir software emulador de terminal serial nos dispositivos
4) configure os parâmetros iguais em cada lado
5) digite comando para gerar trem
